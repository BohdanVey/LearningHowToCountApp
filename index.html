Multinomial theorem

One of the problems you may face is to calculate the number of permutations for some set, which can contain the same symbols. It is a really common problem, which you can see a lot of time in future. Objects in these problems are the length of the set, while classes are a number of different elements. We think that the object in this problem is indistinguishable because if we change the order of object, but object in every position will have the same class, then these permutations will be equal. But classes are distinguishable.

Multinomail theorem problem

Example: Calculate the number of permutations for the letter from word MISSISSIPPI.
Objects: 11, indistinguishable
Classes: 5, distinguishable

Explanation

STEP 1: Calculate number of permutations if each letter werr different$$P(n) =n!$$ STEP 2: Calculate how often each letter appears in the word. $$M = 1$$ $$I = 4$$ $$S = 4$$ $$P = 2$$ STEP 3: Because when we swap the same letters, our permutation remains the sam we
should divide our answer from STEP 1 by number of permutations of each letter.
$$N = {N_{total}\choose{N_{M} N_{I} N_{S} N_{P}}} = {11\choose{1\text{ }4\text{ }4\text{ }2}} = 34650$$

Here you can write your own word and get number of permutation for it

$$N = {N_{total} \choose{N_{I}N_{M}N_{W}N_{S}}} = {11 \choose{4\text{ }1\text{ }2\text{ }4\text{ }}} = 34650$$

To solve problem of this type we often uses Multinomial theorem.

Integer compositions, "Stars and bars"

One of the most challenging and popular problems in classical combinatorics is distributing undistinguished items into distinguishable classes. A wide range of some, on the first glance, totally different, combinatorial problems can be led down to this approach. The most popular example is a problem of the number of compositions of an integer with a certain length. The formulation is following (for wikipedia):

Integer composition problem

A composition of an integer n is a way of writing n as the sum of a sequence of (strictly) positive integers. Two sequences that differ in the order of their terms define different compositions of their sum, while they are considered to define the same partition of that number. A weak composition of an integer n is similar to a composition of n, but allowing terms of the sequence to be zero.undistinguished

And this is literally equivaent to solving the equation: $$x_1 + x_2 + x_3 + ... + x_k = n$$ $$\textrm{Where } x_i, n, k \in Z, n, k \ge 1$$ \(x_i > 0 \textrm{ refers to classical composition, while } x_i \ge 0\) refers to weak composition

Explanation

Let us imagine that undistinguished objects are n integers 1. We have to distribute them among k distinguishable classes (terms of integer composition). Classic compositions refer to case where each class has to contain at least one object (or each term has to be at least one). Weak compositions refer to case where classes can be empty (terms can be zeros).

Another classical problem looks a bit lightweight and corresponds to a classical formulation:

Distributing undistinguished objects among distinguishable classes

How to distribute n similar books to the k students in the class (students are different and unique)?

The answer depends on our ability to leave empty classes or not (as in the case of integer distribution)

However problems about integer composition and book distribution seem to be totally different, they are about literally the same. The solution of classical formulation of the problem (about objects and classes) for n objects and k classes is the following:

Solution

When we don't have an ability to leave empty classes, we have \(C_{n-1}^{k-1}\) variants $$ \textrm{When we have such an ability, then there are } C_{n+k-1}^{k-1} \textrm{ variants}$$ Explanation ("Stars and bars")

Let us allocate all n undistinguished objects ("stars") in a row, now imagine we have k-1 separating "bars", which we need to put somwhere beetwen stars to distribute them among k classes, ranges of stars between neighbour separators refer to corresponding classes (let us say, first class refers to first range from left, etc). So to count the number of distributions we need to find the number of allocations of these k - 1 separators. Since no two separators can be neighbors, we have to choose k - 1 items from n - 1 positions, so we get mentioned earlier answer.

When we can leave empty classes we need just to place in a row n + k - 1 stars and then choose k - 1 star from row to make them separators. Ths counted like selecting k - 1 item from n + k - 1 avilable what refers to earlier mentioned formula

Integer partitions

This is by far the least trivial class of problems. On the first glance, the fact that the classes are indistinguishable changes little to nothing: you only need to divide the result from the case where they are distinguishable by the number of classes. Consider the problem with distributing balls between urns, represented by the equation $$n_{1}+n_{2}+...+n_{k} = M$$ In it, we imagine that the urns are distinguishable, and thus get additional solutions that only differ by which balls are in which urns. By making the urns indistinguishable, we can extract those redundant solutions and only keep the ones that are truly distinct. However, this doesnât change the inherent structure of the problem. All we need to do is to divide the original result by \(k!\). So letâs try to dive a little bit deeper into the subject. What if we donât limit ourselves to a set number of classes? In this case, the result would be all possible ways to distribute \(M\) balls between any number of urns, not just necessarily \(k\). This class can be thought of as attempting to divide a positive integer into parts, without taking the order of the parts into account, like this: $$7=3 + 2 + 1 + 1$$ In number theory, such a process is called partitioning, while the resulting division of the number is called a partition, or an integer partition. The summands of a partition are often called parts. An example of all possible partitions of a number would be: $$4 = 2 + 2 = 3 + 1 = 2 + 1 + 1 = 1 + 1 + 1 + 1$$ Counting the whole number as a partition too (put all 4 balls in one group), the number 4 can be partitioned in 5 different ways. Notice that we donât count \(1 + 3\), for instance, because it only differs from \(3 + 1\) in its order. Because of this, it is a common convention to write the parts in decreasing order. Unfortunately, you will not find a simple way to count the number of partitions for any positive integer N here. This function often denoted as \(p(n)\), does not have a known closed-form expression. However, there is still a lot to learn about it.
Partitions of a number are usually expressed through either a Ferrers diagram or a Young diagram. While I wonât use any of them here, you can look them up to gain a visual intuition on partitions. In particular, they will help you understand a few interesting facts about partitions, such as conjugate partitions and partitions with distinct parts.
Conjugate partitions are created by âflippingâ a diagram of a partition along its main diagonal. Algebraically speaking, for a given partition (example: \(7 = 4 + 2 + 1\)) a conjugate is created by counting the number of parts in a partition, subtracting 1 from each part, and then repeating the process until nothing remains (for the above example, the conjugate would be \(7 = 3 + 2 + 1 + 1\)). Self-conjugate partitions are ones that transform into themselves using this process, like \(6 = 3 + 2 + 1\). Partitions with distinct parts are ones in which no number occurs more than once. These are usually denoted as \(q(n)\), and have their own generating function as well as a recurrence relation given by the pentagonal number theorem. I will not be going to talk about them here though, as they do not relate to the combinatorics in question at all.
\(p_{k}(n)\) is the number of partitions of a number n that contain precisely k parts. This relates back to the question with urns and balls: k is the number of urns. Obviously, $$\sum\limits_{k = 1}^n{p_{k}(n) = p(n)}$$ Since conjugate partitions are constructed using the amount of parts in the original partition, the largest part in a conjugate is always the size of the original. Using this fact, \(p_{k}(n)\) can be expressed as the number of partitions where the largest part is exactly \(k\). By subtracting \(k\), this number is equal to \(p(n-k)\). Now that weâre familiar with the concepts at hand, we can tackle directly calculating p(n). A generating function for p(n) is a reciprocal of the Eulerâs function and looks like this: $$\sum\limits_{n=0}^\infty{p(n)x^n}=\prod\limits_{k = 1}^\infty{\frac{1}{1-x^k}}$$ The asymptotic growth rate of the partition function is given by \(log(p(n)) \approx C\sqrt{n}\text{ }as\text{ }n -> \infty, where\text{ }C = \pi\sqrt{\frac{2}{3}}\).
Finally, we can try to limit both the number of parts as well as the largest part in a partition. Let \(p(N, M, n)\) denote the number of partitions of \(n\) with at most \(M\) parts, each of size at most \(N\). Equivalently, these are the partitions whose Young diagram fits inside an \(MÃN\) rectangle. There is a recurrence relation $$p(N, M, n) = p(N, M-1, n) + p(N-1, M, n-M)$$ obtained by observing that \(p(N, M, n) - p(N, M-1, n)\) counts the partitions of \(n\) into exactly \(M\) parts of size at most \(N\), and subtracting 1 from each part of such a partition yields a partition of \(n â M\) into at most \(M\) parts.
We can see that integer partitions are a broader and a more difficult class of problems than simple combinatorial ones, and that there is a lot to study about them. However, diving deeper into advanced mathematics is always a useful exercise that is worth doing.
